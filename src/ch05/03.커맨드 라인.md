# 03. 커맨드 라인을 통한 입력

`Scanner`클래스의 `nextLine()`외에도 화면을 통해 사용자로부터 값을 입력받을 수 있는 간단한 방법이다.

실행할 프로그램의 `main`메서드가 담긴 클래스의 이름이 `MainTest`라 할 때

```bash
c:\...\ch05\ java MainTest abc 123
```

처럼 커맨드라인을 통해 입력된 두 문자열 "abc", "123"은 String 배열에 담겨<br>
`MainTest`클래스의 `main`메서드의 매개변수(`args`)에 전달된다.

`main`메서드 내에서 `args[0]`, `args[1]`과 같은 방식으로 커맨드라인으로 전달 받은 문자열에 접근할 수 있다.

커맨드라인에 입력된 매개변수는 공백문자로 구분한다.

---

또한 커맨드라인에서 숫자를 입력하여도 숫자가 아닌 문자열로 처리된다는 것을 주의해야 한다.

입력한 숫자로된 문자열을 숫자로 처리하고자 할 경우 아래와 같이 후처리를 필요로 한다.

```java
int num = Integer.parseInt(args[1]); // args[1]은 "123", 123
```

커맨드라인에 매개변수를 입력하지 않을 경우 크기가 0인 배열이 생성되는데<br>

입력된 매개변수가 없다 해서 배열을 생성하지 않을 경우<br>
참조변수 `args`의 값은 `null`이 될 것이고, 배열 `args`를 사용하는 모든 코드에서 에러가 발생할 것이다.

이러한 에러를 피하려면 아래와 같이 조건문을 추가해야 할 것이다.

```java
if (args != null) {
    ...
}
```

⭐ 그러나 입력된 매개변수가 없을 때, JVM이 `null` 대신 크기가 0인 배열을 생성해서<br>
`args`에 전달하도록 구현되어 있어서 이러한 수고를 덜 수 있다.
