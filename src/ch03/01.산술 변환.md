# 01. 산술 변환

이항 연산자의 경우 두 피연산자의 타입이 일치하여야 연산이 가능하다.<br>
대부분의 경우 두 피연산자의 타입 중 큰 타입으로 일치시키는데,<br>
작은 타입으로 형변환을 할 경우 원래의 값이 손실될 가능성이 크기 때문이다.

```java
int i = 10;
float f = 20.0f;

float result = f + (float)i;
float result2 = f + i;
```

위 예시의 `result2`에서는 형변환 연산자를 사용하지 않았다.<br>
그럼에도 가능한 이유는 작은 타입에서 큰 타입으로 형변환하는 경우, 자동적으로 형변환하므로 생략이 가능한 것이다.<br>

⭐ `float`와 `int`는 모두 4Byte지만 수의 표현 범위가 `float`가 크다.

---

위와 같이 타입의 일치를 위해 자동 형변환되는 것을 `산술 변환`이라 하며

이 변환은 이항 연산에서만 아니라 단항 연산에서도 일어난다.

- 두 피연산자의 타입을 같게 일치 (보다 큰 타입으로 일치)
    - long + int -> long + long -> long
    - float + int -> float + float -> float
    - double + int -> double + double -> double

- ⭐ 피연산자의 타입이 int보다 작은 타입이면 int로 변환된다.
    - byte + short -> int + int -> int
    - char + short -> int + int -> int


⭐ 산술 변환이란, 연산 수행 직전에 발생하는 피연산자의 자동 형변환
