# 02. 리터럴

프로그래밍에서 ```상수```란 값을 한 번 저장하면 변경할 수 없는 저장 공간으로 정의하였기에<br>
12, 123, 3.14, 'A'와 같은 일반적 의미의 상수들을 다르게 불러야 했다. 이에 ```리터럴```이라는 용어를 사용한다.

- 변수(variable): 하나의 값을 저장하기 위한 공간
- 상수(constant): 값을 한 번만 저장할 수 있는 공간
- 리터럴(literal): 자체로 값을 의미하는 것

---

```java
boolean b1 = true;
boolean b2 = false;

int i1 = 123;
int i2 = 0b0101;
long l1 = 100L;
long l2 = 0xFFFF_FFFF_FFFF_FFFFL;

float f1 = 3.14;
double d1 = 40.3;
double d2 = 42.5d;
```

위 예시의 모든 변수에 대입된 값들은 리터럴이다.

⭐ 리터럴에 붙은 접미사와 구분자 `_`또한 리터럴의 일부이다.

---

리터럴의 타입은 저장될 변수의 타입과 일치하는 것이 일반적이지만,<br>
타입이 달라도 범위가 넓은 타입에 좁은 타입의 값을 저장하는 것은 허용된다.

```java
int i = 'A';
long l = 123;
double d = 3.14f;
```

`int` 타입 `i`에 `'A'`문자를 초기화하는 경우 문자 'A'의 유니코드인 65가 변수 i에 저장되며<br>
`123`은 접미사가 별도로 존재하지 않아 `int`타입 리터럴이기에 `int`보다 범위가 큰 `long` 타입 `l` 변수에 저장하는 것이 가능하다.<br>
`3.14f`또한 4Byte인 `long` 타입보다 8Byte인 `double` 타입이 큰 범위이기에 변수 `d`에 저장 가능하다.

```java
byte b = 65;
short s = 0x1234;
```

위 예시에서 `byte`는 1Byte이기에, -128 ~ 127 범위 내에 해당하는 `int`타입 리터럴 65가 `byte` 변수 `b`에 할당된다.<br>
`short` 범위 내에 속하는 `int`타입 리터럴 `0x1234` 역시 `s`에 할당 가능하다.

---

```java
System.out.println("1" + "2"); // "12"
System.out.println(true + ""); // "true"
System.out.println('A' + 'B'); // 131
System.out.println('1' + 2); // 51
System.out.println('1' + '2'); // 99
System.out.println('J' + "ava"); // "Java"
System.out.println(true + null); // error
```

`+`연산자를 사용하여 숫자 뿐만 아니라 문자열을 결합할 수 있다.<br>
피연산자 중 어느 한 쪽이 `String`이면 나머지 한 쪽을 `String`으로 변환하여<br>
두 `String`을 결합하는 방식으로 진행된다.

기본형과 참조형의 구분없이 어떠한 타입의 변수도<br>
⭐ 문자열과 덧셈 연산을 수행하면 그 결과가 문자열이 되는 것이다.
