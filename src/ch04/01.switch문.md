# 01. switch문

switch-case 문법은 각 케이스마다 `break`를 설정하여 다른 영역 간 침범을 방지하며 이를 놓칠 경우 큰 문제로 이어질 수 있다.

그러나 경우에 따라서는 고의적으로 `break`문을 생략하여 의도하는 기능을 만들어 낼 수도 있다.

```java
switch (userLevel) {
    case 3 :
        grantDelete();
    case 2 : 
        grantWrite();
    case 1 :
        grantRead();
}
```

위 코드에서는 `userLevel`에 따라 더욱 다양한 권한을 유저에게 부여한다.<br>
`userLevel`이 3일 경우 읽기, 쓰기 및 삭제 권한이 주어지고<br>
`userLevel`이 2일 경우에는 삭제 권한을 제외한 모든 권한이 유저에게 주어진다.

---

`switch`문의 조건식은 결과값이 반드시 정수이어야 하며, 이 값과 일치하는 `case`문의 값 역시 정수이어야 한다.<br>
또한 `case`문의 값은 반드시 상수여야 한다. 변수나 실수, 문자열은 `case`문의 값으로 사용할 수 없다.

- switch문의 조건식 결과는 정수 값(long 타입 제외) 또는 참조형(객체 주소)만 허용
- case문에는 정수 값, 문자 리터럴 또는 참조형만 가능(중복 불가)

---

위 내용까지가 기본적인 `switch`문의 규칙이며 JDK 14부터 switch문의 기능이 확장되어<br>
⭐ 전통적으로 문장이었던 switch문이 식으로도 쓰일 수 있게 되었다.

```java
char grade = switch(score/10) {
    case 9, 10 -> 'A';
    case 8 -> 'B';
    case 7 -> 'C';
    default -> 'F';
};
```

또한 식으로 쓰이는 switch문에서도 기존과 비슷하게 작성하는 것 또한 여전히 가능하다.

```java
char grade = switch(score/10) {
    case 10 :
    case 9 :
        yield 'A';
    case 8 : yield 'B';
    case 7 : yield 'C';
    default : yield 'F';
}
```

다만 `break`문 대신 `yield`문을 사용해서 대입할 값을 지정해야 한다.<br>
`yield`는 `var`처럼 상황에 따라 예약어가 되는 문맥 예약어기에 switch식 내에서만 예약어이다.

---

⭐ `switch`문은 모던 프로그래밍 언어에서 활용도가 높고 점점 복잡해지고 있다.
